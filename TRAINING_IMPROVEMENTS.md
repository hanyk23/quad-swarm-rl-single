# 单无人机雷达避障训练优化记录

本文档整理本轮针对无人机左右摇摆、局部悬停、避障不稳定和微调退化所完成的改进。
内容以当前代码和 `single_quad_obstacles_lidar_body_cbf_v6` 实验为准。

## 0. 版本演进概览

本轮改进不是一次性完成，而是沿着以下实验链逐步定位问题并验证：

| 版本 | 主要变化 | 解决的问题 |
| --- | --- | --- |
| 旧版 | 8 条中心射线 + 1 个全局 clearance，PID 风格避障偏置 | 基线方案，但存在漏检、摆动和策略与控制器对抗 |
| v4 | 9 个小扇区雷达，PID 替换为二维 CBF-QP | 统一第 9 维方向语义，引入硬安全速度约束 |
| v5 | 放宽 CBF 参数，并支持从 v4 直接进入二阶段 | 缓解窄通道无可行速度、过早限速和局部悬停 |
| v6 | 雷达 EMA、CBF 滞回、动作裁剪和独立稳定性微调 | 减少测距跳变、约束抖动、越界动作及微调退化 |

v4、v5 和 v6 使用相同的 9 维雷达观测结构和策略网络输入，因此可以迁移模型权重。
更早版本的雷达维度语义不同，不应直接混用 checkpoint。

## 1. 问题概述

旧模型主要存在以下问题：

- 策略动作方差较大，连续控制指令容易频繁反向，表现为左右摇摆。
- 高斯策略可能采样出动作空间之外的值，导致控制器收到过大的速度指令。
- 旧 PID 风格避障会直接向策略动作叠加修正量，策略和避障器可能相互对抗。
- 雷达距离在相邻时间步波动时，避障约束会频繁启停。
- 单条中心射线可能漏掉位于相邻射线之间的细柱体。
- 从旧 checkpoint 继续训练时继承了动作方差和 Adam 动量，微调初期容易退化。
- 碰撞和速度惩罚突然恢复到完整强度，会让已有策略受到过大的梯度冲击。

### 1.1 v4 训练日志暴露的问题

v4 阶段一在低密度障碍环境中完成约 300 万步训练，最佳 reward 为 `28.826`，且记录到的
障碍物碰撞接近 0。切换到障碍物密度 `0.16`、尺寸 `0.40 m` 的二阶段后，reward 长期
停留在约 `20～24`，任务进度从约 `4.32` 降至 `3.1～3.5`，成功指标也随之下降。

同时，策略损失、KL 和 value loss 没有出现 NaN 或爆炸，说明主要问题不是神经网络训练
失稳，而是二阶段场景难度增加后，安全过滤器限制了有效移动：

- v4 使用 `safe_distance=0.5 m`，两个相对障碍物产生的硬安全区总宽度为 `1.0 m`。
- 场景生成器允许的最小障碍物表面净空为 `0.8 m`。
- 因此部分合法通道在 CBF 约束下不存在同时满足两侧 `0.5 m` 安全距离的速度和位置。
- 多个相邻雷达扇区可能同时约束同一个障碍物，进一步缩小速度可行域。
- CBF 只移除危险速度分量，不主动生成绕行方向；若策略尚未给出横向动作，控制器容易停住。
- QP 无可行解时的保守回退是零水平速度，会直接表现为局部悬停。

这次日志分析促成了 v5 的参数调整，而不是继续单纯增加训练步数。

## 2. 已完成的控制与感知改进

### 2.1 使用机体系观测与机体系速度动作

目标位置、水平速度和雷达方向统一到无人机 yaw 机体系中，策略输出的水平速度再转换到
世界坐标系执行。

这样可以减少机头朝向变化带来的输入语义变化，让策略更容易学习“前、后、左、右”的
稳定控制关系。

### 2.2 将 PID 避障替换为 CBF-QP 安全过滤器

避障层不再向 RL 动作叠加 PID 偏置，而是求解二维速度投影：

```text
minimize  ||v_safe - v_policy||^2
subject to n_i * v_safe <= alpha * (distance_i - safe_distance)
```

其作用是：

- 在没有危险时尽量保持策略原始速度。
- 接近障碍物时只移除朝向障碍物的不安全速度分量。
- 保留仍然安全的横向速度，减少避障器和策略相互拉扯造成的摆动。
- 同时限制水平速度不超过环境允许的最大值。

v4 到 v5/v6 的参数变化：

| 参数 | v4 | v5/v6 | 作用 |
| --- | ---: | ---: | --- |
| `quads_avoid_radius` | 1.2 m | 1.0 m | 缩短 CBF 提前介入距离 |
| `quads_cbf_safe_distance` | 0.5 m | 0.3 m | 减小目标表面安全距离 |
| `quads_cbf_alpha` | 1.5 | 2.0 | 提高安全边界外允许的接近速度 |
| `quads_velocity_yaw_max_speed` | 1.2 m/s | 1.2 m/s | 保持水平速度上限不变 |

在障碍物距离为 `0.8 m` 时，v4 允许的朝向障碍物速度约为
`1.5 * (0.8 - 0.5) = 0.45 m/s`；v5/v6 允许约
`2.0 * (0.8 - 0.3) = 1.0 m/s`。两侧安全距离总和也从 `1.0 m` 降至 `0.6 m`，
小于场景的 `0.8 m` 最小净空，降低了窄通道中因约束过度保守而悬停的概率。

### 2.3 雷达距离 EMA 平滑

CBF 使用指数移动平均后的雷达距离，v6 的更新权重为：

```text
quads_avoid_lidar_filter_alpha = 0.5
```

这能抑制单帧测距跳变，避免安全速度在连续时间步之间突然切换。

### 2.4 CBF 启停滞回

约束在距离小于 `avoid_radius` 时启用，但必须达到
`avoid_radius + hysteresis` 才会解除。

v6 使用：

```text
quads_avoid_activation_hysteresis = 0.04 m
```

它可以避免无人机在避障边界附近反复进入和退出约束状态，从而减少左右抖动。

### 2.5 九方向扇区雷达

原来的 9 维观测由“8 条方向射线 + 1 个无方向 clearance”调整为 9 个等角度方向扇区。
每个扇区：

- 间隔 40 度布置。
- 覆盖 30 度角度范围。
- 均匀采样 5 条子射线。
- 取所有子射线中的最近障碍物或墙面距离。

对应配置：

```text
quads_lidar_sector_angle = 30.0
quads_lidar_sector_samples = 5
```

该改进可以降低细柱体位于两条中心射线之间时的漏检概率，同时让第 9 维也具有明确的
方向语义。

### 2.6 保留低高度保护

CBF 负责水平避障，原有的低高度保护继续独立工作，防止策略在避障过程中持续向地面下降。

## 3. 已完成的策略与训练改进

### 3.1 对策略动作进行硬裁剪

新增 `BoundedActionWrapper`，在动作进入环境前按照 `action_space.low/high` 裁剪。

这解决了高斯策略即使均值经过 `tanh` 限制，采样值仍可能越界的问题，避免异常大的动作
直接传入控制器。

### 3.2 限制策略均值范围

连续动作均值使用：

```text
continuous_tanh_scale = 1.2
```

配合环境入口动作裁剪，使策略输出范围与 1.2 m/s 的速度控制上限保持一致。

### 3.3 重置动作探索方差

v6 从 v5 的最佳模型权重开始，但将可学习动作标准差重置为：

```text
initial_stddev = 0.30
```

训练日志中该值始终约为 `0.299`，说明重置成功。相比旧微调中接近 `0.9` 的方差，
随机动作扰动明显减小。

### 3.4 使用更小的微调学习率和 KL 约束

v6 使用：

```text
learning_rate = 3e-5
kl_loss_coeff = 0.05
```

较小学习率用于降低已有策略被快速破坏的风险，KL 损失用于限制每轮策略更新幅度。
本次训练中 KL 大多约为 `0.001`，没有出现明显的策略突变。

### 3.5 干净地创建 v6 微调 checkpoint

新增 `swarm_rl/bootstrap_lidar_v6.py`，从 v5 best checkpoint 创建独立 v6 实验时：

- 保留模型网络权重。
- 清空 Adam 优化器状态。
- 将优化器学习率重置为 `3e-5`。
- 将动作标准差重置为 `0.30`。
- 将 `train_step` 和 `env_steps` 重置为 0。
- 重置 best reward 记录。
- 不修改原 v5 实验及其 checkpoint。

这样可以继承已有导航能力，同时避免旧优化器动量和高探索方差干扰微调。

### 3.6 碰撞与速度惩罚渐增

v6 不在第一个训练步直接施加完整惩罚，而是从最终权重的 12.5% 开始，在 500 万环境步
内线性恢复到完整值：

```text
anneal_collision_initial_ratio = 0.125
anneal_collision_steps = 5_000_000
```

渐增项包括：

- 无人机碰撞惩罚。
- 平滑碰撞惩罚。
- 障碍物碰撞惩罚。
- 超速惩罚。

这可以减小奖励函数突然变化对旧策略造成的冲击。后期总 reward 下降时，也需要结合原始
成功率、进度和碰撞次数判断，不能只看加权后的总奖励。

## 4. Gymnasium 兼容性修复

`QuadsRewardShapingWrapper.reset()` 已支持：

```python
reset(seed=None, options=None)
```

从而修复以下错误：

```text
TypeError: QuadsRewardShapingWrapper.reset() got an unexpected keyword argument 'seed'
```

底层旧环境仍可维持原来的无参数 `reset()` 调用方式。

## 5. 训练与评估脚本改进

新增或调整了以下入口：

```bash
# 完整重训 v5 的阶段一和阶段二
bash train_single_quad_obstacles_lidar.sh retrain

# 复制 v4 checkpoint，并使用 v5 参数直接训练二阶段
bash train_single_quad_obstacles_lidar.sh stage2-from-v4

# 不加载旧权重，从零直接训练 v5 二阶段
bash train_single_quad_obstacles_lidar.sh stage2-retrain

# 继续已有 v5 二阶段
bash train_single_quad_obstacles_lidar.sh resume

# 从 v5 best 创建 v6，并开始微调
bash train_single_quad_obstacles_lidar.sh

# 继续最新 v6 checkpoint
bash train_single_quad_obstacles_lidar.sh resume-v6

# 测试 v6 最新 checkpoint
bash test_single_quad_obstacles_lidar.sh latest v6

# 测试 v6 reward 最佳 checkpoint
bash test_single_quad_obstacles_lidar.sh best v6
```

`stage2-from-v4` 会复制 v4 的配置和 checkpoint 到独立 v5 目录，再由训练命令覆盖 CBF
参数和二阶段环境配置；原 v4 结果不会被修改。该方式继承策略权重、训练步数和优化器
状态，适合跳过重复的低密度预热。`stage2-retrain` 会覆盖已有 v5 目录并从零训练高密度
场景，通常不如从 v4 初始化稳定。

训练脚本会区分 v4、v5 与 v6 实验目录，防止覆盖原有结果。评估脚本也可以显式选择
`latest/best` 和 `v5/v6`。

## 6. 自动化测试补充

新增测试覆盖：

- 9 个雷达扇区的几何方向和第 9 维方向语义。
- 每个扇区使用多条子射线并返回最近交点。
- 目标和速度向机体系的旋转。
- CBF 在安全边界处阻止朝向障碍物的速度。
- CBF 保留安全的横向速度。
- 第 9 个雷达方向参与安全约束。
- 雷达 EMA 对单帧距离跳变的抑制。
- CBF 启停滞回。
- 动作空间硬裁剪。
- Gymnasium `reset(seed, options)` 兼容。
- v6 checkpoint 的步数、优化器和动作方差重置。
- 惩罚权重从初始比例渐增到最终值。
- 扇区雷达对相邻中心射线之间障碍物的检测。

## 7. v6 训练结果概览

截至 2026 年 6 月 12 日，本次 v6 微调已正常完成：

```text
环境步数：5,001,216 / 5,000,000
最终保存：checkpoint_000004884_5001216.pth
最佳保存：best_000000413_422912_reward_38.975.pth
训练结束状态：正常结束，无 traceback 或 NaN
```

训练期间观察到：

- 动作标准差稳定在约 `0.299`。
- KL 大多约为 `0.001`，策略更新稳定。
- 成功和任务进度指标整体保持稳定，没有出现旧微调中的明显崩溃。
- 原始碰撞次数大致保持在每回合 `0.25～0.32`，仍有继续优化空间。
- 后期 reward 降低的一部分来自碰撞和速度惩罚权重逐渐增大，不等同于任务能力同比下降。

最佳 checkpoint 出现在惩罚权重较低的训练早期，因此最终选型不能只比较 reward。
应在固定随机种子下分别评估 best 和 final 的成功次数、碰撞次数、到达时间及左右速度
反向切换频率。

## 8. 尚未实施的后续工作

按照本轮优化范围，没有继续实施额外的第二步训练策略。后续可在独立实验中考虑：

- 增加动作变化率或 jerk 惩罚，直接约束相邻动作的快速反向。
- 用确定性评估量化左右摇摆，而不是只依赖训练 reward。
- 基于成功率和碰撞率联合保存 checkpoint，避免只按总 reward 选模型。
- 调整最终阶段 CBF 参数或碰撞权重，进一步降低碰撞而不过度牺牲通过率。

这些项目目前仅作为候选方案，不属于本轮已经完成的改动。
